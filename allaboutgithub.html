GitHub Q & A and Definitions

Question Answer
Q1:What is a version control system? A class of systems responsible for managing changes to DOCS, websites or collections of data.
Q2: What is Git? Software for tracking changes in any set of files.
Q3: What is GitHub? A Platform and cloud-based service for development and version control.
Q4: What is the difference between Git and GitHub? Git is a version control software. GitHub is a cloud service that utilizes Git Software.
Q5: Who Started GitHub and how was it started? Tom Preston-Weaver, Chris Wanstrata, P.J Hyett, and Scott Chacon started GitHub in 2007, but rolled it out in 2008 using Ruby on Rails.
Q6: What company owns it now. Microsoft
Q7: How much does a GitHub account cost? 0 USD - 21 USD
Q8: What is the Octocat? The GitHub mascot.
GitHub Terms
  1. Repository: A repository contains all of the project files (including documentation), and stores each file's revision history.
  2. Commit: A commit, or "revision", is an individual change to a file (or set of files).
  3. Fork: A fork is a personal copy of another user's repository that lives on your account. Forks allow you to freely make changes to a project without affecting the original repository.
  4. Push: To push means to send your committed changes to a remote repository on GitHub.com.
  5. Pull Requests: Pull requests are proposed changes to a repository submitted by a user and accepted or rejected by a repository's collaborators.
  6. Workflows: A GitHub workflow is a set of automated steps that you can add to your repository to perform tasks such as building, testing, packaging, releasing, or deploying a project.
  7. Issues: Issues are suggested improvements, tasks or questions related to the repository.
  8. Raw Button: The raw button on GitHub is a feature that allows you to view or download the raw content of a file without any formatting or styling.
  9. Blame Button: The blame command is a Git feature, designed to help you determine who made changes to a file.
